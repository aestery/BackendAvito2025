## Сервис назначения ревьюеров для Pull Request’ов

### Как запустить
Для запуска необходимо выполнить команду ```docker-compose up``` соответственно иметь установленными docker и git, чтобы клонировать репозиторий. Работоспособность проверялась на WSL:UBUNTU.

### Нагрузочное тестирование
Для нагрузочного тестирования использовался ```locust 2.42.5``` соответственно, чтобы запустить сценарии необходимо иметь его установленным. 

Для тестирования на соответствие условиям задачи были выставленны параметры `number of users = 5` и `runp up = 1` для достижения целевой RPS = 5.

Ошибки обрабатываемые сервисом рассматриваются как успешное выполнение его вункции, в failure могут попадать только необрабатываемые ошибки.

Не все сценарии хорошо соответствуют действительной потенциальной нагрузке (особенно это касается сценария переназаначения пользователя), а их вероятностное распределение оценено "на глаз" в данной версии тестирования.
